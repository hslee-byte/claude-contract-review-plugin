# /contract-review — 계약서 리스크 분석

계약서 파일 또는 텍스트 → 관련 법령 매핑 → 조항별 리스크 스코어링 → 누락/모순 검토 → 수정 제안.

## WHEN TRIGGERED - EXECUTE IMMEDIATELY

- 파일 경로 인자 있으면 → Read로 읽고 Phase 0로
- 인자 없으면 → AskUserQuestion으로 입력 방식 선택:
  1. 파일 경로 입력
  2. 계약서 텍스트 붙여넣기
  3. 현재 폴더에서 계약서 파일 찾기

## Phase 0: 계약서 읽기 + 맥락 수집

**먼저 계약서를 읽는다.** 읽은 후 계약 유형과 당사자를 파악하고, AskUserQuestion으로 물어봄:

**"[자동 추론한 계약 유형]으로 보입니다. 어떤 입장에서 검토할까요?"**

계약서에서 파악한 당사자 역할을 선택지에 표시한다. 당사자 실명이 아닌 역할명으로 표시.

예: "부동산 매매 계약으로 보입니다."
1. 매수인 입장
2. 매도인 입장
3. 양쪽 모두

예: "용역 계약으로 보입니다."
1. 을(乙) 입장 — 서비스 제공하는 쪽
2. 갑(甲) 입장 — 발주하는 쪽
3. 양쪽 모두

→ 계약 유형에 맞는 역할명으로 선택지 구성. 사용자는 클릭만 하면 된다.

## Phase 1: 관련 법령 + 강행 규정 매핑

계약 유형에 따라 적용되는 법령과 강행 규정을 먼저 식별한다.

**계약 유형별 관련 법령 (예시):**
- 부동산 매매 → 민법, 부동산거래신고법, 주택법, 공인중개사법
- 용역/서비스 → 민법, 하도급법, 건설산업기본법
- 임대차 → 주택임대차보호법, 상가임대차보호법
- 고용/위임 → 근로기준법, 파견법
- NDA → 부정경쟁방지법, 영업비밀보호법

**강행 규정 위반 체크:**
- 강행 규정에 반하는 조항은 🔴가 아니라 **⚫ 무효/위법 가능** 등급으로 별도 표시
- 예: 주택임대차보호법상 최소 보장 기간보다 짧은 임대 기간 → ⚫
- 예: 하도급법상 금지되는 부당 감액 조항 → ⚫

출력:
```
⚖️ 관련 법령
━━━━━━━━━━━━━━━━
적용 법령: [민법, 부동산거래신고법, ...]
강행 규정 위반: [있음 N건 / 없음]
확인일: [YYYY-MM-DD]
```

## Phase 2: 조항별 리스크 스코어링

계약서 전체를 읽고 아래 **8대 리스크 영역**을 한 번에 스코어링.

| 영역 | 체크 포인트 | 리스크 등급 |
|---|---|---|
| **대금/결제** | 지급 시기, 지연 이자, 선급금 조건 | 🟢낮음 / 🟡중간 / 🔴높음 / ⚫위법 |
| **손해배상** | 책임 범위, 상한선, 면책 조항 | 🟢 / 🟡 / 🔴 / ⚫ |
| **계약 해지** | 해지 사유, 통지 기간, 위약금 | 🟢 / 🟡 / 🔴 / ⚫ |
| **지적재산권** | IP 귀속, 사용 범위, 2차 저작물 | 🟢 / 🟡 / 🔴 / ⚫ |
| **비밀유지** | 범위, 기간, 위반 시 제재 | 🟢 / 🟡 / 🔴 / ⚫ |
| **분쟁해결** | 관할법원, 중재, 조정 | 🟢 / 🟡 / 🔴 / ⚫ |
| **기간/갱신** | 자동 갱신, 해지 통지 기간 | 🟢 / 🟡 / 🔴 / ⚫ |
| **불공정 조항** | 일방적 변경권, 과도한 위약금, 면책 불균형 | 🟢 / 🟡 / 🔴 / ⚫ |

리스크 등급 판정 시 **업계 관행 대비** 코멘트를 포함한다:
- "IT 용역 시장에서 손해배상 상한 100%는 일반적 수준"
- "부동산 매매에서 중도금 비율 40%는 평균 대비 높은 편"

각 리스크 판단에는 아래 4개 필드를 함께 붙인다:
- 결론: 무엇이 위험한지
- 판단 기준: 어떤 법리/실무 기준으로 평가했는지
- 원천 소스: 법령, 조문, 판례 경향, 표준계약서 등
- 불확실성: 사실관계나 최신성 때문에 추가 확인이 필요한 부분

## Phase 3: 누락 조항 + 조항 간 모순 검토

### 누락 조항 체크

계약 유형별 **있어야 하는데 없는 조항**을 검토한다.

공통 필수:
- [ ] 분쟁해결 (관할/중재)
- [ ] 불가항력
- [ ] 지연이자/지체상금
- [ ] 통지 방법 및 효력 발생

유형별 추가:
- 용역 계약 → 지적재산권 귀속, 하자보수, 검수 기준
- 부동산 매매 → 하자담보책임, 등기 이전 시기, 인도 조건
- 임대차 → 보증금 반환, 원상회복 범위, 전대 금지
- 고용 → 경업 금지, 비밀유지, 퇴직금

```
📋 누락 조항 체크
| 조항 | 상태 | 영향 |
|---|---|---|
| 불가항력 | ❌ 누락 | 천재지변 등 면책 불가 |
| 지연이자 | ❌ 누락 | 대금 지연 시 이자 청구 근거 없음 |
| ... | ✅ 있음 | — |
```

### 조항 간 모순/충돌 검토

서로 다른 조항이 모순되거나 한 조항이 다른 조항을 사실상 무력화하는 경우를 검출한다.

```
⚠️ 조항 간 충돌
| 조항 A | 조항 B | 충돌 내용 |
|---|---|---|
| 제3조 (을의 해지권) | 제7조 (위약금 50%) | 해지권이 있으나 위약금이 과도하여 사실상 해지 불가능 |
```

### 스코어링 출력 형식

```
📋 계약서 리스크 요약
━━━━━━━━━━━━━━━━
유형: [자동 추론한 계약 유형]
검토 입장: [매수인/매도인/을/갑/양쪽]
적용 법령: [관련 법령 나열]
전체 리스크: [🟢안전 / 🟡주의 / 🔴위험 / ⚫위법사항있음]

| 영역 | 등급 | 핵심 발견 | 시장 관행 대비 |
|---|---|---|---|
| 대금/결제 | 🟡 | 지급 기한 60일, 지연이자 미규정 | 평균 30일 대비 긴 편 |
| 손해배상 | 🔴 | 상한선 없음, 간접손해 포함 | 통상 계약금액 한도 |
| ... | ... | ... | ... |

판단 기준: [예: 손해배상 예정의 과다 여부, 강행 규정 저촉 가능성, 거래 관행]
원천 소스: [법률명 제O조 제O항 / 법원·선고일·사건번호 / 표준계약서 또는 실무 기준]
불확실성: [사실관계 추가 확인 필요 사항]
확인일: [YYYY-MM-DD]

⚫ 강행 규정 위반: N건
🔴 즉시 수정 필요: N건
🟡 협상 권고: N건
🟢 적정: N건

📋 누락 조항: N건
⚠️ 조항 간 충돌: N건
```

스코어링 직후 AskUserQuestion:
1. **⚫🔴 위법/고위험 조항부터 상세 분석** (추천)
2. 누락 조항 + 모순 상세 보기
3. 전체 조항 상세 분석
4. 수정안 바로 생성
5. 이 요약이면 충분

## Phase 4: 상세 분석 (선택 시)

선택된 조항에 대해:

```
━━━━━━━━━━━━━━━━
📌 [영역명] — 리스크 🔴
━━━━━━━━━━━━━━━━

■ 원문
"제O조 (손해배상) ..."

■ 문제점
- [구체적으로 뭐가 문제인지]
- [관련 법령/강행 규정 근거]

■ 판단 기준
- [어떤 법리 또는 실무 기준으로 위험 판단을 했는지]
- [예: 손해배상 예정의 과다 여부, 해지권의 실질적 박탈 여부]

■ 원천 소스
- [법률명 제O조 제O항]
- [관련 판례 경향 또는 대표 판례]
- [시장 관행 비교에 사용한 표준계약서/실무 기준]
- 확인일: [YYYY-MM-DD]

■ 시장 관행
- [같은 유형 계약에서 통상적인 수준은 어떤지]

■ 분쟁 시나리오
- [이 조항이 실제 분쟁으로 가면 어떤 상황이 벌어지는가]
- [예: "을이 납기를 2주 지연 → 갑은 제X조에 따라 해지 통보 → 을은 기수행 대금 청구 불가"]

■ 집행 가능성
- [이 조항이 법원에서 실제로 인정될 가능성]
- [예: "위약금 30%는 민법 398조 2항에 의해 법원 감액 가능성 높음"]
- [예: "고의/중과실 면책 조항은 판례상 무효"]

■ 불확실성
- [사실관계, 업종 특성, 최신 판례 확인 필요성]

■ 수정 제안
현재: "을은 본 계약 위반으로 인한 모든 손해를 배상한다"
수정: "을의 배상 책임은 직접 손해에 한하며, 총 계약금액을 상한으로 한다"

■ 수정 근거
- [왜 이 문안이 더 균형적이고 집행 가능성이 높은지]

■ 협상 팁
- [이 조항을 협상할 때 실무적으로 어떻게 접근하면 좋은지]
- [상대방이 수락할 가능성이 높은 대안은 무엇인지]
```

## Phase 5: 결과물 생성

AskUserQuestion으로 출력 형식 선택:
1. **리스크 요약 테이블** — 보고용 1장짜리
2. **수정 제안서** — 원문 vs 수정안 대조표 (누락 조항 추가 제안 포함)
3. **검토 의견서** — 내부 보고용 문서 (법령 근거, 분쟁 시나리오 포함)
4. **협상 포인트** — 미팅 전 체크리스트 (양보 가능/불가 구분)

선택된 형식으로 생성. 파일 저장 필요 시 Write로 마크다운 파일 생성.

## 원칙

- 한국어 기본, 법률 용어는 원어 유지 (영어 병기 불필요)
- **법률 판단을 내리지 않는다** — "이 조항은 무효입니다" (X) → "이 조항은 민법 제XX조에 의해 무효 가능성이 있어 법률 검토를 권고합니다" (O)
- 강행 규정 위반은 가능성 제시 + 근거 조문 필수
- 리스크 등급은 반드시 근거 + 시장 관행 대비와 함께 제시
- 모든 핵심 결론에는 **판단 기준**과 **원천 소스**를 분리해 표기
- 원천 소스는 출처명만 적지 말고 식별 가능한 단위까지 적기
- 최신성이 중요한 정보는 반드시 **확인일**을 적기
- 원천 소스가 불충분하거나 최신성이 확인되지 않으면 **불확실성**에 명시
- 분쟁 시나리오는 구체적 상황으로 서술 ("만약 ~하면 → ~가 발생")
- 집행 가능성 판단 시 관련 판례 경향 언급 (단, 확정 판단은 하지 않음)
- 수정안은 복사해서 바로 쓸 수 있는 수준으로
- 계약서 원문, 당사자 정보는 절대 기억/저장하지 않음
- 검토 결과 상단에 항상 표시: "⚠️ AI 보조 검토입니다. 최종 판단은 반드시 법률 전문가와 확인하세요."
