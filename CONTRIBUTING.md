# Contributing

Thanks for your interest in improving this project.

This repository is open source, but not every contribution will be merged. The maintainer keeps final control over scope, quality, and release direction.

## What This Project Optimizes For

- professional usefulness
- source traceability
- low-friction default UX
- concise outputs first, depth on demand
- safer wording in legal and medical contexts

## Preferred Contributions

- better source attribution
- clearer reasoning standards
- lighter default flows
- better Lite-mode output quality
- better uncertainty handling
- more precise practical templates
- typo fixes and documentation improvements

## Contributions Likely To Be Rejected

- changes that make the default flow much heavier
- extra questions that are not necessary for a useful first-pass result
- more confident or absolute legal/medical conclusions without stronger source support
- flashy feature additions that increase maintenance cost without clear workflow value
- changes that reduce traceability of output

## Before You Open A PR

Please explain:

1. what problem you are solving
2. why this change improves the workflow
3. whether it makes Lite mode heavier
4. whether it improves or weakens source traceability

## Review Principles

- Lite mode should stay fast and readable.
- Deep mode can be richer, but should stay structured.
- In high-risk domains, uncertain claims should stay explicitly uncertain.
- Maintainability matters as much as feature breadth.

## Small Practical Rule

If a proposal adds complexity, it should also clearly reduce user effort, error risk, or verification time.
